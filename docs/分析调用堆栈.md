# 分析调用堆栈

## 1. 大盘复盘（Market Review）

入口路径有三条：API 触发、CLI `--market-review`、定时任务 `run_full_analysis()`。

```
═══════════════════════════════════════════════════════════════
 入口 A：API 触发
═══════════════════════════════════════════════════════════════
POST /api/v1/analysis/market-review
  │
  ├─ _compute_market_review_override_region(config)        # api/v1/endpoints/analysis.py
  │    ├─ get_open_markets_today()                         # src/core/trading_calendar.py
  │    │    └─ is_market_open(mkt, today)                  # exchange-calendars 判断
  │    └─ compute_effective_region(region, open)           # 裁剪出有效市场
  │         ↓ 返回 "" → 直接返回"非交易日" ❌
  │
  ├─ _try_acquire_market_review_lock(config)               # 进程锁 + 文件锁
  │
  └─ submit_background_task(_run_market_review_background)
       │
       ├─ _build_market_review_runtime(config)             # 组装 notifier + analyzer + search
       │    ├─ NotificationService(source_message)
       │    ├─ SearchService(bocha_keys, tavily_keys, ...)  # 仅在 has_search_capability_enabled() 时
       │    └─ GeminiAnalyzer(config)                       # 仅在有 LLM API key 时
       │
       ├─ run_market_review(override_region)               # src/core/market_review.py
       │    └─ MarketAnalyzer(search_service, analyzer, region)  # src/market_analyzer.py
       │         └─ run_daily_review()
       │              ├─ get_market_overview()              # 指数 + 统计 + 板块
       │              │    ├─ _get_main_indices()
       │              │    │    └─ DataFetcherManager.get_main_indices(region)  # data_provider/base.py
       │              │    ├─ _get_market_statistics()      # [CN only] 涨跌统计
       │              │    │    └─ DataFetcherManager.get_market_stats()
       │              │    └─ _get_sector_rankings()        # [CN only] 板块排名
       │              │         └─ DataFetcherManager.get_sector_rankings(5)
       │              │
       │              ├─ search_market_news()               # 搜索引擎抓新闻
       │              │    └─ SearchService.search_stock_news("market", ...)
       │              │         └─ [provider failover: Anspire → Bocha → Tavily → Brave → ...]
       │              │
       │              └─ generate_market_review(overview, news)  # LLM 生成报告
       │                   ├─ _build_review_prompt()        # 拼 prompt（含 MarketProfile + StrategyBlueprint）
       │                   │    ├─ MarketProfile            # 区域元数据（news_queries, has_market_stats 等）
       │                   │    └─ MarketStrategyBlueprint  # 策略框架（趋势/流动性/板块维度）
       │                   ├─ analyzer.generate_text(prompt, max_tokens=8192, temperature=0.7)
       │                   │    └─ _call_litellm()          # LiteLLM Router → Gemini/Anthropic/OpenAI
       │                   └─ _inject_data_into_review()    # 将数据表注入 LLM 输出的 markdown
       │
       ├─ notifier.save_report_to_file()                   # 写 reports/market_review_YYYYMMDD_HH.md
       ├─ notifier.send()                                  # 推送到通知渠道
       └─ db.save_analysis_history()                       # 写数据库（report_type="market_review"）


═══════════════════════════════════════════════════════════════
 入口 B：CLI --market-review（仅大盘复盘）
═══════════════════════════════════════════════════════════════
main()
  │
  ├─ parse_arguments() → args.market_review == True
  │
  ├─ [交易日过滤]
  │    ├─ get_open_markets_today()
  │    └─ compute_effective_region() → "" 则跳过 ❌
  │
  ├─ build_market_review_runtime(config)                  # src/core/market_review_runtime.py
  │    ├─ NotificationService()
  │    ├─ SearchService(...)
  │    └─ GeminiAnalyzer(config)
  │
  └─ _run_market_review_with_shared_lock(config, run_market_review, ...)
       ├─ try_acquire_market_review_lock(config)           # src/core/market_review_lock.py
       │    └─ threading.Lock + fcntl.flock（跨进程防重）
       ├─ run_market_review(...)                           # 同入口 A 的 run_market_review
       └─ release_market_review_lock()                     # finally 块释放


═══════════════════════════════════════════════════════════════
 入口 C：定时任务 run_full_analysis()（个股 + 大盘）
═══════════════════════════════════════════════════════════════
main()
  │
  ├─ parse_arguments() → args.schedule 或 config.schedule_enabled
  │
  └─ run_full_analysis(config, args, stock_codes)          # main.py
       │
       ├─ _compute_trading_day_filter(config, args, codes) # 按市场过滤非交易日股票
       │
       ├─ StockAnalysisPipeline(config)                    # 初始化服务（见下方个股分析）
       │
       ├─ pipeline.run(stock_codes)                        # ① 个股分析（见下方）
       │
       ├─ [analysis_delay 秒间隔]
       │
       └─ _run_market_review_with_shared_lock(             # ② 大盘复盘（复用 pipeline 的 notifier/analyzer/search）
            config, run_market_review,
            notifier=pipeline.notifier,
            analyzer=pipeline.analyzer,
            search_service=pipeline.search_service,
            merge_notification=True,                       # 合并模式：跳过单独推送
            override_region=effective_region,
          )
```

## 3. 多Agent编排模式（Multi-Agent Orchestrator）

当 `config.agent_mode` 启用或 `config.agent_skills` 配置了具体技能时，个股分析走 Agent 路径。若同时 `agent_arch=multi`，则进入 **多 Agent 编排器（AgentOrchestrator）**，由多个 Specialist Agent 并行/串行协作完成分析。

以下以 `mode=specialist`（生产环境默认）为例，展开完整调用堆栈。

### 3.1 架构总览

       │

#### 3.3.3 _build_agent_chain() — 基础链 [orchestrator.py:584-616](src/agent/orchestrator.py#L584-L616)

按 `mode` 返回不同的 agent 序列：

| mode | 链结构 | 说明 |
|------|--------|------|
| `quick` | `[TechnicalAgent, DecisionAgent]` | 仅技术面，跳过情报/风险 |

## 4. 大盘 vs 个股：关键差异

| 维度 | 个股分析 | 大盘复盘 |
|------|---------|---------|
| 入口 | `StockAnalysisPipeline.run()` | `run_market_review()` |
| 数据源 | 单股 OHLCV + 实时行情 + 筹码分布 | 全市场指数 + 涨跌统计 + 板块排名 |
| 搜索 | 多维度（新闻/风险/业绩/行业）× 每只股票 | 市场级新闻查询（按 region） |
| LLM 调用 | `analyzer.analyze()` → 结构化 JSON（AnalysisResult） | `analyzer.generate_text()` → 自由格式 markdown |
| 输出 | `AnalysisResult`（评分/建议/仪表盘） | 原始 markdown 报告文本 |
| 并发 | ThreadPoolExecutor（默认 3 线程） | 顺序执行（多市场逐个处理） |
| 通知 | 单股推送 or 汇总推送 | 独立推送 or 合并到个股报告 |
